# Update Coverage Groups Tool

This tool manages the `functions-missing-coverage-grouped` files by comparing them with the latest coverage results to:

1. Remove functions that are now covered
2. Append new uncovered functions for later analysis

## Purpose

During the fuzzing coverage improvement cycle, functions that were previously uncovered may become covered in subsequent runs. This tool ensures that the grouped coverage files reflect the current state by:

- Removing functions from `functions-missing-coverage-grouped` that now have coverage
- Adding new uncovered functions discovered in the latest run
- Preserving the `analysis` field for functions that remain uncovered

## Usage

### Command Line

```bash
update-coverage-groups [--magic-dir PATH] [--return-json] [--verbose]
```

### Options

- `--magic-dir PATH`: Directory containing coverage files (default: `magic`)
- `--return-json`: Output result as JSON to stdout (for workflow integration)
- `-v, --verbose`: Enable verbose output for debugging

### Workflow Integration

This tool runs after Step 29 in the `workflow-fuzzing-coverage.json` workflow:

```json
{
  "name": "Step 30: Update Coverage Groups",
  "type": "task",
  "description": "Update grouped coverage file by removing now-covered functions",
  "prompt": "update-coverage-groups --return-json",
  "model": { "type": "PROGRAM", "model": "inherit" },
  "output": {
    "capture": true,
    "save_to": "magic/functions-missing-covg-grouped-latest.json"
  }
}
```

## File Formats

### Input Files

1. **Latest Coverage File**: `magic/functions-missing-covg-{timestamp}.json`
   - Generated by `covg-eval` and `covg-scoring` tools
   - Contains current uncovered functions

2. **Previous Grouped File**: `magic/functions-missing-covg-grouped-{timestamp}.json`
   - Contains grouped functions with analysis from previous runs

### Output File

**Updated Grouped File**: `magic/functions-missing-covg-grouped-{new_timestamp}.json`
- Retains still-uncovered functions with their existing `analysis` fields
- Appends new uncovered functions without `analysis` (to be added by coverage-phase-4)
- Updates summary statistics

## Algorithm

The tool uses a combination of function signature and uncovered line ranges to uniquely identify functions:

```
key = function::contract::source_file::line_range1|line_range2|...
```

This ensures accurate matching even when the same function has different uncovered sections between runs.

cd tools/update_coverage_groups

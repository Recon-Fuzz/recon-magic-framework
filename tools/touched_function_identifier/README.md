# Touched Function Identifier

A Python CLI tool that analyzes `sol-expand` output to identify all functions touched by target functions in a smart contract project.

## Overview

This tool processes the call trees generated by `sol-expand` and creates a comprehensive list of all functions that are directly or indirectly called by your target functions. It automatically filters out:
- Interface contracts
- Library contracts
- Proxy contracts (ERC1967, UUPS, Transparent, etc.)
- Reentrancy guards
- Authorization/Access control contracts (Ownable, AccessControl, etc.)

## Installation

No external dependencies required! Uses only Python standard library.

**Requirements**: Python 3.6+

## Usage

### Basic Usage

```bash
python touched_function_identifier.py \
  --sol-expand-dir morpho-blue/context_output \
  --target-functions magic/target-functions.json
```

This will generate `./magic/functions-to-cover.json` with all touched functions.

### Custom Output Location

```bash
python touched_function_identifier.py \
  --sol-expand-dir morpho-blue/context_output \
  --target-functions magic/target-functions.json \
  --output custom/path/output.json
```

### Arguments

- `--sol-expand-dir` (required): Directory containing the sol-expand output (e.g., `morpho-blue/context_output`)
- `--target-functions` (required): Path to the `target-functions.json` file
- `--output` (optional): Output file path (default: `./magic/functions-to-cover.json`)

## Input Format

### target-functions.json

The input file should contain an array of contracts and their target functions:

```json
[
  {
    "contract": "Morpho",
    "target_functions": [
      "accrueInterest",
      "borrow",
      "supply"
    ]
  }
]
```

### sol-expand Output

The tool expects the standard `sol-expand` output structure with function markdown files containing call trees:

```
context_output/
├── src/
│   └── Morpho.sol/
│       ├── function_accrueInterest_struct_MarketParams.md
│       ├── function_borrow_struct_MarketParams_uint256_uint256_address_address.md
│       └── ...
└── ...
```

## Output Format

### functions-to-cover.json

The tool generates a JSON file mapping each contract to its touched functions:

```json
{
  "Morpho": {
    "functions_to_cover": [
      "_accrueInterest",
      "_isHealthy",
      "accrueInterest",
      "borrow",
      "supply",
      "withdraw"
    ]
  }
}
```

Functions are automatically:
- Deduplicated
- Sorted alphabetically
- Filtered to exclude interfaces, libraries, and common infrastructure contracts

## Filtering Logic

The tool automatically excludes:

### Contract Types
- **Interfaces**: Contracts declared with `interface ContractName { ... }`
- **Libraries**: Contracts declared with `library ContractName { ... }`

### Infrastructure Contracts

**OpenZeppelin:**
- Proxies: `ERC1967Proxy`, `TransparentUpgradeableProxy`, `UUPSUpgradeable`, `BeaconProxy`, `Proxy`, etc.
- Reentrancy Guards: `ReentrancyGuard`, `ReentrancyGuardUpgradeable`
- Authorization: `Ownable`, `Ownable2Step`, `AccessControl`, `AccessControlEnumerable`, etc.

**Solmate:**
- Authorization: `Auth`, `Owned`, `RolesAuthority`, `MultiRolesAuthority`
- Reentrancy Guards: `ReentrancyGuard`

**Solady:**
- Authorization: `Ownable`, `OwnableRoles`
- Reentrancy Guards: `ReentrancyGuard`, `ReentrancyGuardTransient`

## Implementation Details

### Call Tree Parsing

The tool parses call trees from sol-expand markdown files using regex to extract function calls in the format:

```
FUNCTION: ContractName.functionName(args...)
```

### Contract Type Detection

The tool searches the source directory for contract definitions and checks their declaration:
1. Prioritizes `src/` and `contracts/` directories
2. Excludes build artifacts (`out/`, `build/`, `artifacts/`, `cache/`)
3. Reads the source file to identify `interface`, `library`, or `contract` declarations
4. Caches results to avoid redundant file reads

### Performance

- Uses caching for contract type detection
- Processes files in parallel-friendly structure
- Handles large codebases efficiently

## Troubleshooting

### Warning: Could not find function file

If you see warnings like:
```
⚠ Warning: Could not find function file for Morpho.accrueInterest
```

Check that:
1. The function name matches exactly (case-sensitive)
2. The sol-expand output includes the target contract
3. The function exists in the contract

### Warning: Could not read contract file

If you see warnings about reading contract files, the tool will default to treating them as regular contracts (safest option). This usually happens when:
1. The contract is defined in a dependency (node_modules, lib/)
2. The contract is in a non-standard location

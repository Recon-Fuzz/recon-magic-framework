**IMPORTANT**: the ONLY objective of this phase is to generate a pre-audit preparation file that breaks down the codebase strucure in `audit-prep.md`.

<instructions>
## Step 1 - Compilation
Compile the codebase with:

```
forge build --build-info
```

### Error Handling

#### Not Using Foundry
If the code doesn't compile determine if the code is not using foundry.

**IMPORTANT**: If the code is not using foundry stop immediately with a warning.
- Generate a `CRITICAL_STOP.MD` file saying that you're going to stop because the code is incompatible with this workflow.

#### Using Foundry
1. If the codebase is using Foundry: 
    - resolve compilation errors
    - run a fresh compilation (run `forge clean` then `forge build --build-info`)
    - identify the file in the `build-info` directory.

Identify the output directory, as well as the build info file (it's a json file inside the `build-info` directory which is located in the outpu) ex: `out/build-info/<hash-based-name>.json`

## Step 2 - Context Creation

1. Generate the context information for this smart contract system by running:
```
sol-expand --extract-context PATH_TO_BUILD_INFO_FILE.json
```

2. Identify where the context generated by the `sol-expand` tool was saved
3. Add this information to a file called: `magic/audit-prep.md`
4. **IMPORTANT**: exclusively output the following message to the terminal: "Here's the path for the context for the audit: <path/to/audit-prep.md>"
</instructions>

<output>
## Success Criteria

2. No other information is output to the terminal or generated regarding the content of the contracts.
